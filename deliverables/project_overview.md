# Project Overview: E-commerce Event Streaming Pipeline

## Components

### 1. **Data Generator (`data_generator.py`)**: 
- Generates synthetic e-commerce events (e.g., `purchase`, `view`, `add_to_cart`).
- Each event contains the following fields:
   - `event_key`: Unique identifier for the event.
   - `user_id`: Randomly generated user ID.
   - `product_id`: Randomly selected product ID.
   - `event_type`: Event type (e.g., `purchase`).
   - `timestamp`: Event timestamp.

### 2. **Spark Structured Streaming (`spark_streaming_to_postgres.py`)**:
- Streams the CSV data generated by `data_generator.py` using Spark Structured Streaming.
- For each incoming CSV file:
   - Reads data using Spark’s `readStream`.
   - Performs transformations (e.g., converts timestamp, assigns UUIDs).
   - Writes data in micro-batches to PostgreSQL using the `foreachBatch` sink.

### 3. **PostgreSQL**:
- Data is ingested into the `ecommerce_events` table in PostgreSQL.
- The table contains fields like `event_key`, `user_id`, `product_id`, `event_type`, and `timestamp`.
- A view `recent_purchases` is created for querying the latest purchase events.

## Data Flow

1. **Data Generation**:
   - `data_generator.py` generates CSV files every 5 seconds, simulating e-commerce events.

2. **Data Streaming**:
   - `spark_streaming_to_postgres.py` continuously reads these files and streams them to PostgreSQL.

3. **PostgreSQL Storage**:
   - Event data is stored in the `ecommerce_events` table in PostgreSQL.

## Technologies Used

- **Apache Spark**: Real-time data processing with Structured Streaming.
- **PostgreSQL**: Data storage and querying.
- **Python**: Data generation, streaming logic, and system orchestration.
